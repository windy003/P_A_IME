# PyIME —— Windows 11 简易中文拼音输入法

纯 Python 标准库实现(无任何第三方依赖),约 600 行单文件。
原理:全局键盘钩子(`WH_KEYBOARD_LL`)拦截按键 → 内部组拼音、查词库 →
tkinter 候选窗 → `SendInput(KEYEVENTF_UNICODE)` 把汉字直接发到当前程序。
对记事本、浏览器、VS Code、终端等任意程序有效。

## 启动 / 退出

```
python pyime.py          # 带控制台日志启动
启动输入法.bat            # 后台静默启动
```

退出:任意时刻按 **Ctrl+Shift+Q**。

> ⚠️ 使用前请把 Windows 系统输入法切到**英文键盘**(或英文模式),
> 否则系统自带的微软拼音会先截走字母键,两个输入法会"打架"。

## 按键

| 按键 | 功能 |
|---|---|
| Ctrl+Space 或 单击 Shift | 中 / 英 模式切换 |
| a-z | 输入拼音(支持整句词组,如 `shurufa` → 输入法) |
| `'` | 手动隔音(如 `xi'an` → 西安) |
| 1-9 | 选择对应候选 |
| 空格 | 选第 1 个候选 |
| `-` `=` / PgUp PgDn | 候选翻页 |
| Backspace | 删一个字母 |
| Esc | 取消本次输入 |
| Enter | 把拼音字母原样上屏 |
| , 。 ; : ? ! " 【 】 《 》 等 | 中文模式下自动转全角标点(引号自动配对) |

支持不完整音节前缀匹配:打 `zhon` 也能出"中"。
选了词组前半部分后,剩余拼音会留在缓冲区继续组词(如 `nihaoshijie` →
选"你好"后继续出"世界")。

支持**模糊音**:默认开启 z/zh、c/ch、s/sh、n/l、an/ang、en/eng、in/ing、
ian/iang、uan/uang(如打 `zongguo` 也能出"中国",`gaoxin` 也能出"高兴");
精确匹配的候选始终排在模糊匹配之前。要增删模糊音对,编辑 `pyime.py`
顶部的 `FUZZY_PAIRS` 列表即可(f/h、l/r 已写好,取消注释即生效;
清空列表则完全关闭模糊音)。

支持**简拼**:每个音节只打声母即可,如 `zg`→这个/中国、`bj`→北京、
`nh`→你好、`rmb`→人民币(翘舌 zh/ch/sh 自动按 z/c/s 处理)。当你输入的
字母**全部是声母**时触发,候选按词频排序;打全拼时不受影响。可在 `pyime.py`
顶部把 `ENABLE_SHOUPIN` 设为 `False` 关闭。

## 文件

| 文件 | 说明 |
|---|---|
| `pyime.py` | 输入法主程序 |
| `pinyin_simp.dict.yaml` | 词库(RIME 简体拼音词库,6.5 万词条,带词频) |
| `启动输入法.bat` | 静默启动脚本 |
| `test_e2e.py` / `test_target.py` | 端到端自动化测试(模拟按键打"你好") |
| `pinyin.txt` | 单字拼音原始数据(备用,程序未使用) |

## 测试

```
python pyime.py --selftest   # 拼音切分 / 查词逻辑测试
python test_e2e.py           # 端到端:模拟敲 nihao+空格,验证目标窗口收到"你好"
```

## 已知局限

- 这是"键盘钩子式"输入法,不是系统级 TSF 输入法:不会出现在系统输入法列表里,
  没有候选窗跟随光标的原生精度(取不到插入符时跟随鼠标),管理员权限窗口里无效
  (除非以管理员运行)。要做成真正的系统输入法需用 C++ 实现 TSF
  (ITfTextInputProcessor)并注册 COM 组件,工程量大一个数量级。
- 无词频学习、无用户词库;候选排序固定按词库词频。
- 词库版权:RIME pinyin_simp(源自 Android 开源项目拼音输入法,Apache 2.0)。
